02. Töltsd ki a hiányzó részeket!
-----------------------------------
a) Xerox Chad:

   valtoztat(chad);
   // chad értéke még mindig: 5
   (Magyarázat: érték szerint adtuk át, csak a másolat változott 999-re)

   if (n == 0) return 0;  // base case

03. Melyik Boss Karakter jelenik meg?
--------------------------------------
a) int* p = nullptr;
   printf("%d", *p);
   → SEGFAULT SAMU jelenik meg.
   Nullptr-t dereferencelünk — érvénytelen memóriacímhez nyúlunk,
   a program azonnal crashel.

b) int* p = (int*)malloc(sizeof(int));
   *p = 42;
   // free() soha nem hívódik
   → MEMORY LEAK MIKI jelenik meg.
   A malloc()-kal foglalt memóriát sosem szabadítjuk fel free()-vel,
   így az a program futása alatt elvész — memóriaszivárgás keletkezik.

c) int vegtelen(int n) {
       return vegtelen(n);  // base case?
   }
   → STACK OVERFLOW STAN jelenik meg.
   Nincs base case, a függvény végtelen mélységig hívja saját magát,
   a call stack betelik és a program összeomlik.


04. Saját Brainrot Karakter
-----------------------------
Név:       Const Konstantina
Fogalom:   const kulcsszó — megváltoztathatatlan érték
Személyiség:
   Konstantina egyszer dönt, és soha többé nem változtat. Határozott,
   kiszámítható, és ha valaki megpróbálja módosítani az értékét —
   a fordító azonnal visszautasítja. Nem rugalmas, de megbízható.
   Ő az egyetlen, akiben mindig bízhatsz.

Idézet:
   "Én nem változom. Ez nem gyengeség — ez karakter."

Kódpélda:
   const int MAXERTEK = 100;
   MAXERTEK = 200;  // FORDÍTÁSI HIBA — Konstantinát nem lehet módosítani!

   const int* p = &MAXERTEK;  // pointer const int-re: *p nem írható
   // p = &masik;             // de maga a pointer átírható
